18. auto title
一股“我他妈受够了”的怒火混合着技术宅的倔强直冲天灵盖。林橙猛地站起来，异色的双瞳同时亮起冰蓝与金黄的光芒——“Dual-Core Debugging”模式启动。

视野瞬间被数据化。整个房间变成了由线条和节点构成的模型。她无视了余胜军惊讶的目光，将全部精神集中在那一堆该死的表格上。她拿起一份被标注为“完美样本”的表格（唯一一份被管理局审核通过的），用Azure Debugger（左眼冰蓝）对其进行高速扫描分析，解析其结构、字段、填写规范。

紧接着，Golden Bug（金黄右眼）锁定目标——那堆需要填写的空白和填错的表格（stupid_forms）：

// 假设已定义完美模板结构体 Form 和 模板实例 perfect_template
extern Form stupid_forms[MAX_FORMS]; // 指向那堆待处理表格的“数组”
extern int num_forms; // 表格数量

for (int i = 0; i < num_forms; i++) {
    // Step 1: 粗暴初始化 - 将当前垃圾表格的内存内容直接覆盖为完美模板的副本
    memcpy(&stupid_forms[i], &perfect_template, sizeof(Form));

    // Step 2: 唯一需要个性化设置的地方 - 申请人ID (假设林橙的ID是常量)
    stupid_forms[i].applicant_id = LING_CHENG_ID;
    // (其他字段如日期等，在完美模板中已预设为合理默认值或占位符，林橙认为无需修改)
}

意念指令如同洪流般发出。

嗡…

一股无形的能量场以林橙为中心扩散开来。桌面上，那些空白或填错的纸质表格无风自动，纸张哗啦啦地翻飞。上面的字迹如同被无形的橡皮擦抹去，又在同一时间，被无形的笔以绝对工整、完全一致的格式重新书写填满！打印机（连接着电脑，处理电子表格）更是像被打了鸡血，发出疯狂的嗡鸣，吐纸口以惊人的速度喷吐出雪片般的、已经完美填写打印好的电子表格实体文件！

短短十几秒，风停纸落。

桌面上，原本杂乱无章的“垃圾堆”，变成了一摞摞高度整齐、字迹工整规范、填写内容完全符合要求的完美表格！每一份都像是从同一个模子里刻出来的，连签名都仿得一模一样（林橙直接copy了完美样本上的签名图像数据）。

“卧槽！”余胜军看得目瞪口呆，下巴都快掉地上了，“批量内存覆写？！直接操作物理信息态？！这效率…这精准度…神器啊！这要是用在自动化产线调试、固件批量刷写…”技术宅的思维瞬间飞到了工业应用的星辰大海。

林橙长舒一口气，看着整洁的桌面，一种久违的、debug成功的清爽感油然而生。虽然能量消耗不小，但比起精神折磨，这点消耗值了！

然而，余胜军那声“卧槽”和他闪闪发光的眼神，如同黑暗中的灯塔，瞬间吸引了测试中心其他几个同样饱受文书之苦的倒霉蛋。他们探头探脑地过来，看到林橙桌上那堆“神迹”般的完美表格，眼睛都直了。

“林…林橙同志？”一个看起来挺面善的中年大姐小心翼翼地问，“那个…能不能…帮我们也…‘处理’一下？”她指了指自己手里厚厚一叠待填的报表，脸上写满了祈求。

“对对对！帮帮忙吧！回头请你吃饭！”
“我这有几份紧急的，下午就要交！”
“还有我的！我的！”

很快，小小的休息区围过来四五个人，手里都捧着或多或少的待处理文件，眼神热切地看着林橙，如同看着救世主。甚至有人偷偷塞过来零食和饮料。

林橙看着这些同病相怜的“难友”，又看了看自己刚刚“解放”的双手，一种莫名的“侠义感”（或者说，对自动化脚本的炫耀心理）涌上心头。// 反正脚本写好了，多处理几份也只是增加循环次数。

她萝莉的小手一挥：“放那儿吧！排队！”

她来者不拒，将大家的文件堆在一起，再次开启“双核调试”模式。意念中简单地修改了一下脚本：
num_forms += new_forms_count; // 增加待处理表格数量
stupid_forms = realloc(stupid_forms, num_forms * sizeof(Form)); // (意念中动态扩展数组指针指向新加入的文件堆)

指令再次发出！能量波动比刚才更明显一些。

哗啦啦——嗡——

熟悉的景象再现。所有的文件，无论纸质电子，都在几秒内被“格式化”并重新“写入”，变得完美无瑕、整齐划一。

“成了！”
“太感谢了！”
“林橙你真是天才！”
“这能力太实用了！”

众人欢呼雀跃，拿着焕然一新的文件，对林橙感激涕零。余胜军在一旁看得啧啧称奇，记录着能量波动的特征（在他感知中）。

林橙享受着这短暂的、如同超级工具人般的成就感，疲惫感似乎都减轻了。她拿起手边一瓶刚“赚来”的饮料喝了一口。

然而，致命的疏忽往往发生在最松懈的时刻。

在那一堆被送来的文件中，混进了一份不起眼的、封面标记着“Confidential - Level B”的蓝色文件夹。里面是一份需要林橙本人签署的《关于S级波动源关联事项的附加保密协议及承诺书（修订版）》（confidentiality_agreement_v2_final_signed.docx）。这份文件因为涉及更高密级，格式和内容与其他表格有细微差别，且需要手写签名和日期。

林橙刚才扫描的“完美样本”，是一份普通的D级表格。而她在批量处理时，为了省事，粗暴地使用memcpy进行内存覆盖，并且没有仔细检查stupid_forms数组中每一个元素的“类型”和源地址！她的脚本指针，在覆盖到那份蓝色保密协议时，鬼使神差地…发生了极其微小但致命的偏移。

这份保密协议旁边，余胜军之前随手放了一个他自己的草稿本。本子上摊开的那一页，正好是他绞尽脑汁写了一半、准备送给心仪女孩（管理局文印室的一位姑娘）的情书草稿！上面写着诸如“你的笑容如同春风拂过我的代码，消除了所有冗余的循环…”，“每次看到你，我的心脏就像遭遇了硬件中断，跳动的频率溢出…”之类充满技术宅浪漫（？）的肉麻句子，末尾还有一个画了一半的爱心。

林橙脚本中的memcpy源地址指针（&perfect_template），在覆盖到保密协议对应的“内存位置”时，不知是因为能量扰动、文件堆叠的物理偏移，还是单纯的世界恶意…竟然跳转指向了余胜军那张情书草稿所在的位置！

于是，灾难发生了。
