16.
郑峰的行动效率极高。在姐妹俩搬进公寓的第三天，林橙就接到了通知，要求她前往管理局下属的专项测试中心进行所谓的“能力可控性与基础应用评估”。通知措辞官方而冰冷，带着不容置疑的意味。

测试中心位于管理局总部大楼的地下深层，环境充满了科幻感。纯白色的走廊，自动滑行的感应门，穿着制服的工作人员行色匆匆。林橙被带到一间宽敞的多功能测试场，地面是某种高弹性的吸能材料，墙壁上布满了各种传感器探头。空气中弥漫着淡淡的臭氧和金属的味道。

郑峰亲自到场监督。他穿着笔挺的制服，双手抱胸站在观察窗前，眼神锐利如常，像在审视一件需要严格质检的精密仪器。林小雨则被安排在隔壁的观察室，隔着单向玻璃可以看清场内情况。

“林橙，”郑峰的声音通过场内广播传来，清晰而平稳，“今天主要是基础体能和神经反应测试，目的是建立你的生理基线数据，并为后续可能的能力应用提供参考。请配合工作人员指示。”

林橙看着场边穿着白大褂的技术员拿着平板电脑准备记录，又看看场地中央的跑步机、测力计、反应速度测试仪等设备，内心充满了程序员对重复性体力劳动的深深厌倦。// 又是这些无聊的体力活…消耗卡路里，产出毫无意义的数据点。

第一个项目是耐力跑。林橙站上跑步机，随着速度逐渐提升，她娇小的身体开始感到吃力。呼吸变得急促，小腿肌肉传来熟悉的酸痛感。汗水顺着额角流下，沾湿了纯白的鬓发。// PAIN_THRESHOLD reached. Sensory feedback flooding… 她脑中闪过一行注释。

枯燥、疲惫、不适…负面感受开始堆积。一个念头不受控制地在她脑中浮现：既然自己的能力能操作现实，那能不能…稍微“优化”一下自身感官，让这个过程不那么痛苦？比如，降低点痛觉阈值，或者加快点神经反应让动作更协调省力？这应该比操作外部物体容易吧？

这个想法如同伊甸园的苹果，充满了诱惑力。在又一次迈开沉重的步伐时，林橙偷偷开启了右眼的“Golden Bug”模式。淡金色的光芒在右瞳中流转。她小心翼翼地用意念构建指令，试图进行微调：

#define PAIN_THRESHOLD 0.7f // 将痛觉阈值降低到原来的70%，理论上痛感减轻
#pragma optimize("sensory_input", speed) // 优化神经信号传输速度，提升反应

指令输入完毕的瞬间，一股远超预期的能量波动猛地从她体内爆发出来！测试场内明亮的灯光剧烈地闪烁、明灭不定，发出滋滋的电流声。墙壁上的传感器读数瞬间飙红，警报声短促地响了一下又被技术员手忙脚乱地关闭。

林橙只觉得大脑“嗡”的一声，仿佛被强行塞入了一个超频运转的CPU。优化…失控了！

痛觉阈值确实降低了，跑步带来的肌肉酸痛感明显减弱。但副作用来得更猛烈。

脚下跑步机传送带的细微纹理，此刻在她感知中如同粗糙的砂纸，每一次落脚都带来强烈的摩擦感，甚至能感觉到材料微米级的凹凸不平。身上吸汗的运动服面料，从柔软的棉质变成了布满微小毛刺的砂布，不断摩擦着皮肤。更可怕的是，几米开外，郑峰制服上的金属纽扣、肩章、腰带扣…它们散发出的冰冷质感，如同实质的寒气针，隔着空气刺向她裸露的皮肤，让她瞬间起了一层鸡皮疙瘩，不由自主地打了个寒颤。

汗水流到唇边，咸味被放大了十倍，如同吞咽了一口浓缩海水。林小雨在观察室里隔着玻璃递给她一瓶水（通过场内的传物口）。林橙喝了一口，清澈的矿泉水在她口中瞬间变成了浓缩的柠檬汁混合着铁锈的味道，酸涩刺激得她眼泪差点掉下来，猛地咳嗽起来。

跑步机电机运转的低沉嗡鸣被放大成持续的噪音轰炸。传感器微弱的电流声清晰可闻。观察室玻璃上林小雨担忧的脸庞，在她眼中细节被过度锐化，甚至能看到对方脸上微小的绒毛和毛孔，视觉信息量过大导致一阵眩晕。

“呃…！”林橙闷哼一声，脚步踉跄，差点从跑步机上摔下来。她感觉自己像一个被强行安装了超高灵敏度传感器的机器人，各种冗余的、放大的信号疯狂涌入，塞满了她的处理核心。这根本不是优化，是感官灾难。

“林橙！停止能力！报告状态！”郑峰严厉的声音透过广播传来，带着一丝不易察觉的紧张。他紧紧盯着场内传感器反馈的混乱数据流。

林橙强忍着不适，集中起几乎要溃散的精神。必须立刻终止这个失控的进程！她用意念疯狂输入：

#include "sensory_default.h" // 紧急载入默认感官配置文件
killall -9 sensory_optimization_daemon // 强制终止所有优化进程

指令生效。又一股能量波动，但比刚才弱一些。疯狂涌入的超量感官信息如同潮水般迅速退去。脚下恢复了正常的触感，口中那股可怕的酸涩味也消失了，郑峰那边传来的“金属寒意”也感觉不到了。世界恢复了“正常”的分辨率。

代价是巨大的疲惫感。林橙大口喘着粗气，汗水浸透了运动服，小脸苍白，直接瘫坐在了跑步机旁边，仿佛刚经历了一场高强度的精神鏖战。

测试被迫暂停。工作人员紧张地检查设备和林橙的状态。郑峰脸色铁青地翻看着平板电脑上刚才记录下的、如同癫痫发作般的生理数据曲线。

林小雨被允许进入场内。她跑到林橙身边，递上一条毛巾和一瓶新的水（林橙这次喝得小心翼翼，确认是正常的矿泉水才敢大口喝），满脸担忧：“姐，你没事吧？刚才吓死我了！灯光乱闪，你表情好痛苦！”

林橙虚弱地摆摆手：“…没事。玩脱了…想偷个懒，结果系统崩了…”她自嘲地扯了扯嘴角。

郑峰走了过来，居高临下地看着坐在地上的林橙，眼神复杂。他没有立刻责备，而是沉声问：“你刚才试图做什么？”

“…尝试微调自身感官阈值和神经传导效率，降低测试不适感。”林橙老实承认，声音还有点发虚，“显然…参数设错了，或者权限不足，引发了连锁崩溃。”她用程序员的方式解释着失败。

郑峰沉默了几秒，似乎在消化这个答案。他看着林橙疲惫的样子，又看了看平板上那些混乱但峰值惊人的能量读数，最终没有苛责。“休息十五分钟。补充能量。”他指了指场边准备好的能量饮料和高热量食物。

休息区。林橙小口吃着能量棒，试图补充消耗。林小雨坐在旁边，递给她一杯刚冲好的速溶咖啡，上面浮着一层薄薄的奶泡。“喝点热的，缓缓神。”

林橙接过温热的纸杯，看着杯口那圈白色的奶泡，劫后余生的疲惫感中，一丝属于程序员的、对“可控性”的不服输念头又冒了出来。刚才的优化失控是灾难，但能力的本质似乎是对“信息”和“状态”的操作？那操作一个具体的、微小的、无害的对象呢？

她盯着那圈奶泡，心念微动。这一次，她没有开启Golden Bug模式，只是集中起温和的意念，仿佛在脑中轻声输入一行简单的格式化输出指令：
fprintf(coffee_foam, "cat"); // 尝试在咖啡奶泡上‘打印’出一只猫的图案

指令发出，能量消耗微乎其微。只见杯口那层薄薄的奶泡，如同被无形的磁极引导，开始缓缓流动、凝聚。几秒钟后，一个歪歪扭扭、线条简单但神韵十足的猫头轮廓，清晰地浮现在了咖啡表面。

“哇！！！”林小雨第一个发现，惊喜地叫出声来，“猫！姐！是猫！你弄的？”她凑近了看，眼睛闪闪发亮。

这声惊呼也吸引了旁边的人。一个身影几乎是瞬移般地出现在了休息区——正是隔壁的技术宅邻居余胜军。他不知何时也来到了测试中心（可能是作为技术支援或纯粹围观），此刻正瞪大了眼睛，死死盯着林橙手里的咖啡杯。

“猫…咖啡拉花…意念成型？！”余胜军的声音因为激动而有些变调，“这…这是微观流体控制？还是直接的信息态物质重组？能量消耗效率如何？精度控制…天呐！这可比3D打印牛逼多了！完全无接触、无模具！”他语速飞快，唾沫横飞，恨不得立刻掏出笔记本记录。

林橙被他的突然出现和连珠炮似的问题吓了一跳，手一抖，咖啡差点洒出来，那只奶泡小猫也随之微微变形。她有些窘迫地把杯子往回收了收：“…就…随便试试。”她可不想再被这技术狂魔缠上。

郑峰也走了过来，目光扫过那只歪歪扭扭的咖啡猫，又落在林橙略显尴尬的脸上，最后看向自己手中平板电脑上那份写满了“异常”、“峰值”、“不稳定”、“潜力巨大”、“生活化应用？”等关键词的初步测试报告。

他揉了揉发胀的太阳穴，似乎感到一阵头痛。沉默了几秒钟，他做出了结论，声音里带着一种放弃治疗般的无奈：

“基础体能数据…参考价值有限。能力实战评级，暂无法确定，能量消耗模式极端非线性，且存在高度不可控风险。”他顿了顿，目光再次扫过那只咖啡猫，“但…在特定生活辅助细节操作方面，展现出…独特且难以复制的潜力。综合评定：极不稳定，需长期观察与严格管控。今日测试…到此为止。”

他把平板递给旁边的记录员，转身离开，背影看起来比平时沉重了一分。留下林橙看着杯子里快消散的奶泡猫，林小雨一脸的崇拜，以及余胜军眼中燃烧的、仿佛发现了新大陆般的熊熊之火。
